Systems</b> (<a href=\"./cs5460/index.html\">Spring 2014</a> at University of Utah)\n\t\t</div><p></p>\n\n                <p>\t\n\t\t</p><div id=\"menu_caption\"><b><a name=\"current_research\">Current research</a></b></div>\n\t\t<p></p>\n\n\t\t<p>\n\t\t</p><div id=\"menu_item\"><b>RedLeaf: Verified Operating Systems in Rust</b>\n\n\t\tRedLeaf is a new operating system, and associated formal verification tools for\n\t\timplementing provably secure and reliable systems in the Rust programming\n\t\tlanguage. RedLeaf brings together state-of-the-art results from verification,\n\t\tprogramming-language, and systems research communities in order to enable\n\t\tunprecedented security and reliability guarantees in low-level systems\n\t\tsoftware. To achieve complete verification of the entire software stack, i.e.,\n\t\toperating system and applications, the RedLeaf team will develop a set of new\n\t\ttools, a collection of techniques and engineering disciplines, and a\n\t\tmethodology focused on rapid development of verified systems software. The\n\t\tRedLeaf OS will run on an embedded CPU of a medical sensor, implement a network\n\t\tfunction virtualization framework aimed at line-rate network processing, and\n\t\tprovide a general platform for a broad range of verifiably secure systems. \n\n\n\t\t</div>\n\t\t<div id=\"menu_item\">\n                        <a href=\"https://www.nsf.gov/awardsearch/showAward?AWD_ID=1837127&amp;HistoricalAwards=false\">NSF abstract</a>\n\t\t</div> \n\t\t<p></p> \n\n                <p> \n\n                </p><div id=\"menu_item\"><b>Redshift: An Operating System for Pervasive Hardware Acceleration</b>\n\n\t\tIn contrast to today's systems centered around general-purpose processors also\n\t\tknown as central processing units (CPUs), the next generation of\n\t\thigh-performance computers will inherently rely on diverse, heterogeneous\n\t\thardware ranging from many-core processors like Intel Xeon Phi that contains up\n\t\tto 72 processor cores and graphical processing units (GPUs) to specialized\n\t\thardware accelerators, like specialized machine-learning chips and\n\t\tfield-programmable gate arrays (FPGAs) re-programmed on demand for a specific\n\t\ttask. In a hardware-accelerated environment that consists of many diverse\n\t\texecution units, the execution of a program is no longer a conventional thread\n\t\ttied to a single CPU, but a graph of small computations scheduled on a set of\n\t\thardware accelerators each implementing a part of the program logic. Redshift\n\t\tis a new operating system for developing applications that leverage performance\n\t\tof a heterogeneous hardware-accelerated system.\n\n\t\tAt the core of Redshift is a dataflow programming model that enables execution\n\t\tof commodity programs on a network of heterogeneous hardware execution units\n\t\twith only minimal modifications. Redshift implements programs as collections of\n\t\tasynchronous invocations that transparently move execution between hardware\n\t\tfunctions. A novel runtime maps computations to execution units, balances load\n\t\tamong them, and scales the hardware graph of computation in response to load.\n\n\t\t</div> \n\n               <div id=\"menu_item\"> <a href=\"https://www.nsf.gov/awardsearch/showAward?AWD_ID=1817120&amp;HistoricalAwards=false\">NSF abstract</a> </div> \n               <p></p> \n\n\t\t<p> \n\n                </p><div id=\"menu_item\"><b>Horizon: Secure Large-Scale Scientific Cloud Computing</b>\n\n\t\tHorizon is a novel cloud architecture aimed at providing data and computation\n\t\tsecurity within a scientific cloud. Horizon builds upon three premises: (1)\n\t\tstrong isolation on end-hosts, (2) fine-grained isolation in the cloud network,\n\t\tand (3) cloud-wide information flow control. To protect the end-hosts, Horizon\n\t\tdevelops a new layered hypervisor, and disaggregated virtualization stack with\n\t\tkey features of: language safety, software fault isolation, and integrated\n\t\tsoftware verification. To provide secure cloud network environment, Horizon\n\t\trelies on a new network architecture and implements a distributed network\n\t\tfirewall, where all network communication and exchange of rights are mediated\n\t\tand controlled by the rules of the object capability system. To protect the\n\t\tcloud data, Horizon develops a set of abstractions and mechanisms to enforce\n\t\tcloud-wide information flow control. In Horizon all data is labeled. The\n\t\thypervisor mediates all communication of each virtual machine and enforces\n\t\tpropagation of labels and security checks for each cloud computation.\n\n\t\t</div> \n\n               <div id=\"menu_item\"> <a href=\"https://www.nsf.gov/awardsearch/showAward?AWD_ID=1840197&amp;HistoricalAwards=false\">NSF abstract</a> </div> \n               <p></p> \n\n\t\t<p> \n\n                </p><div id=\"menu_item\"><b>Secure and Oblivious Data Bases with secure computation and SGX</b>\n\t\t\n\t\tWe collaborate with crypto and DB teams to develop novel data base architectures that \n                protect sensitive data through novel cryptographic primitives and secure enclaves (Intel SGX). \n\n\t\t</div> \n\n               <div id=\"menu_item\"> <a href=\"https://nsf.gov/awardsearch/showAward?AWD_ID=1817143&amp;HistoricalAwards=false\">NSF abstract</a> </div> \n               <p></p> \n\n\n\t\t<p>\n\t\t</p><div id=\"menu_item\"><b>LCDs: Disaggregated System Services\n\t\t\t\tThrough Lightweight Capability\n\t\t\t\tDomains.</b>\n\n\t\t\tFor several decades the operating system community has\n\t\t\taccepted that the only way of minimizing the trusted\n\t\t\tcomputing base, and constructing more secure, least\n\t\t\tauthority systems, is to reimplement monolithic kernel\n\t\t\tfunctionality as a set of isolated microkernel servers.\n\t\t\tInstead of splitting the core functionality of a\n\t\t\ttraditional operating system into multiple servers, we\n\t\t\tdevelop a mechanism that can securely isolate\n\t\t\tindividual kernel components right inside the address\n\t\t\tspace of the monolithic operating system kernel.\n\t\t\tEffectively, we build a capability machine for the\n\t\t\tLinux kernel. \n\t\t</div>\n\t\t<div id=\"menu_item\">\n                        <a href=\"http://www.cs.utah.edu/~aburtsev/lcd-doc/\">LCDs source code and documentation</a>\n\t\t</div> \n\t\t<div id=\"menu_item\">\n project page</a>\n\t\t</div> \n\n\t\t<p></p> \n\n                <div id=\"menu_item\"><b>CapNet: Capability-Enabled Networks and Clouds.</b>\n\t\t\tBuilding on the principles of capability access control, CapNet\n\t\t\trepresents system resources as a graph of objects connected\n\t\t\twith edges that represent rights, or capabilities. By controlling\n\t\t\tthe distribution of capabilities, CapNet (i) implements\n\t\t\tstrong, fine-grained isolation of network hosts; (ii) develops\n\t\t\tmechanisms for decentralized, application-driven dynamic\n\t\t\tmanagement of connectivity; and (iii) provides a formal foundation\n\t\t\tfor establishing secure collaboration in face of dynamic,\n\t\t\tmistrusting principals and insecure end-hosts.\n\t\t</div>\n\n\t\t<div id=\"menu_item\">\n                        <a href=\"https://www.flux.utah.edu/project/CapNet\">CapNet project page</a>\n\t\t</div> \n\t\t<p></p>\n\n                <p>\n\t\t</p><div id=\"menu_item\"><b>Fast cross-core communicaiton for non-uniform memory access (NUMA) architectures</b>\n\n\t\t\tWe work on designing a new efficient communication mechanisms aimed at providing efficient \n                        communication on modern NUMA machines. \n\n                </div>\n\t\t<p></p>\n\n\t\t <p>\t\n\t\t</p><div id=\"menu_caption\"><b>Past projects</b></div>\n\t\t<p></p>\n\n\t\t<p></p><div id=\"menu_item\"><b>Deker: Decomposing Commodity Kernels for Verification.</b>\n\t\tDeker is a framework for decomposing and verifying commodity operating system kernels. \n\t\tDeker turns a de-facto standard commodity operating system kernel into a \n\t\tcollection of strongly isolated subsystems suitable for verification. \n\t\tDespite multiple decades of evolution and improvements in software verification \n\t\ttools, almost none of them made their way into regular industry practice. \n\t\tDeker aims to amend this using a holistic approach unifying modular redesign \n\t\tof legacy components with customized verification techniques. While \n\t\tdecomposing the kernel and providing complete isolation of subsystems, \n\t\tDeker remains practical: retains source-level compatibility with the \n\t\tnon-decomposed kernel, enables incremental adoption, and remains fast. \n\t\t\n\t   \t</div>\n\t\t<div id=\"menu_item\">\n                        <a href=\"https://www.flux.utah.edu/project/deker\">Deker project page</a>\n\t\t</div>\n\t\t<p></p> \n\n\n\t\t<p>\n\t\t</p><div id=\"menu_item\"><b>XCap: Practical Capabilities and Least Authority\t\t\n\t\t\t\tfor Virtualized Environments.</b> \n\t\t\tXCap is a secure environment for least-authority\n\t\t\texecution of applications and system services.\n\t\t\tUnmodified, untrusted, off-the-shelf applications,\n\t\t\trunning on untrusted operating systems, are isolated by\n\t\t\ta virtual machine monitor. XCap's default -- a share\n\t\t\tnothing environment -- is augmented by a capability\n\t\t\taccess control model: a clean and general abstraction,\n\t\t\tenabling fine-grained delegation of rights in a\n\t\t\tflexible and manageable way.  </div>\n\n\t\t<div id=\"menu_item\">\n                        <a href=\"https://www.flux.utah.edu/project/xcap\">XCap project page</a>\n\t\t</div> \n\t\t<p></p>\n\n\n\t\t<!--\n\t\t<P>\n\t\t<DIV ID=\"menu_item\"><B>LibOS: An operating system stack for\n\t\t\t\tlow-latency datacenter communication.</B>\n\n\t\t\tlibOS is a new operating system mechanism designed to\n\t\t\tprovide applications with a low-latency access to\n\t\t\tperipheral devices.</DIV>\n\n\t\t</P>-->\n\n\t\n\n\t\t<p>\n\t\t</p><div id=\"menu_item\"><b>XenTT: Deterministic system analysis.</b>\n\t\t\n\t\t\tXenTT is full-system deterministic replay engine for Xen capable of capturing and \n\t\t\treplaying execution of Xen VMs. XenTT creates a \n\t\t\tfoundation for a deterministic replay analysis platform aimed at automatic debugging, and \n\t\t\tanalysis of complex software systems.
