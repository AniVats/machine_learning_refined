     <h2 class=\"hp-main-title\">Yonghwi Kwon<font color=\"#AAA\">, the University of Virginia</font> </h2>\n                          <p>\n                            I am the John Knight Career Enhancement Assistant Professor in the Department of Computer Science at the University of Virginia. I received my Ph.D. in Computer Science at Purdue University. My research interests include software systems security (i.e., securing software to prevent cyberattacks), cyber forensics (i.e., recovering cyber forensic evidence from data/programs), and software engineering (software testing and reverse-engineering). \n                            My research aims to build systems that automatically analyze, understand, and enhance software systems.\n                          </p>\n                          <p>\n                            I have been honored with NSF CRII Award (2019), ACM SIGPLAN Distinguished Paper Award (2019), Maurice H.         <div class=\"ci-text\">\n                              <h4>Active Defense against Cyber Attacks</h4>\n                              <p>Vulnerable programs are targeted by cyber attackers. Manually patching the vulnerabilities cannot keep up with the fast growing cyber attack trend.\n\n                                We develop automated techniques that analyze insecure programs and make them secure <a href=\"data/a2c_ndss17.pdf\" target=\"blank\">[NDSS'17]</a><a href=\"data/pad_ase17.pdf\" target=\"_blank\">[ASE'17]</a> leveraging compiler-based infrastructure (e.g., <a href=\"https://llvm.org/\" target=\"_blank\">LLVM</a>).\n                              </p>\n                            </div>\n                          </div>\n\n                          <!-- Item 2 -->\n                          <div class=\"info-block-w-icon\">\n                            <div class=\"ci-icon\">\n                              <i class=\"lnr lnr-link\"></i>\n                            </div>\n                            <div class=\"ci-text\">\n                              <h4>Cyber Forensic against Cyber Attacks</h4>\n                              <p>After systems are hacked, understanding details of the attack such as how it happened (i.e., the initial compromise), what is the intention, and who is behind the attack is important.\n\n                                We develop precise information-flow tracking techniques for forensic analysis <a href=\"data/ldx_asplos16.pdf\" target=\"blank\">[ASPLOS'16]</a><a href=\"data/mci_ndss18.pdf\" target=\"_blank\">[NDSS'18]</a> to uncover such details.\n                              </p>\n\n                            </div>\n                          </div>\n                          <!-- class=\"section-title-block second-style\">\n                  <h2 class=\"section-title\">Research</h2>\n                  <h5 class=\"section-description\">Solving System Security Problems via Fundamental Program Analysis</h5>\n                </div>\n\n                <div class=\"section-content\">\n                  <div class=\"row\">\n                    <div class=\"col-xs-12 col-sm-12\">\n                      <!-- research Content -->\n                      <div class=\"research-content\">\n\t\t\t\t\t\t<p>\n\t\t\t\t\t\t\t<font color=\"black\" size=\"3em\">\n\t\t\t\t\t\t\t<b>I build systems that can analyze and prevent sophisticated cyber attacks such as Advanced Persistent Threats (e.g., Stuxnet). </b>\n\t\t\t\t\t\t\t</font>\n\t\t\t\t\t\t</p>\n\t\t\t\t\t\t<p>\n\t\t\t\t\t\t\t<font color=\"black\" size=\"3em\">\n\t\t\t\t\t\t\t\tSpecifically, (1) I build systems that harden existing programs to prevent <a href=\"https://en.wikipedia.org/wiki/Advanced_persistent_threat\" target=\"_blank\">advanced attacks</a> (e.g., <a href=\"https://en.wikipedia.org/wiki/Zero-day_(computing)\" target=\"_blank\">zero-day exploits</a>). (2) I build reverse-engineering techniques that uncover malicious behaviors (e.g., compromising systems and <a href=\"https://en.wikipedia.org/wiki/Data_breach\" target=\"_blank\">data leak</a>) of sophisticated malware on various architectures (e.g., x86, ARM, MIPS) with little to no knowledge of the malware. (3) my research recovers forensic evidence of sophisticated attacks to reveal attack paths (e.g., how an attack happened, who are the attackers, and what are the ramifications of the attack).<br>\n\t\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\t\t<a href=\"#publications\">My list of publications is maintained here</a> and my research has been generously supported by NSF (National Science Foundation), ONR (Office of Naval Research) through IAI (Inteligent Automation, Inc), and NVIDIA. \n\t\t\t\t\t\t\t</font>\n\t\t\t\t\t\t</p>\n\n\t\t\t\t\t\t<p align=\"center\">\n\t\t\t\t\t\t</p><hr width=\"300\">\n\t\t\t\t\t\t<p></p>\n\t\t\t\t\t\t<p>\n\t\t\t\t\t\t\t<b><font color=\"black\" size=\"4em\"><i class=\"fas fa-shield-alt\"></i> Preventing Malicious Payload Injection Attacks</font></b><br>\n\t\t\t\t\t\t\t<img src=\"images/research/shellcode.jpg\" style=\"border-radius: 8px;\" width=\"210\" id=\"flimg\">\n\t\t\t\t\t\t\tMalicious attackers (remotely) inject malicious payloads into the victim systems in order to achieve their goals such as disrupting systems and exfiltrating secret information.\n\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\t<b>\n\t\t\t\t\t\t\tMy research aims to proactively prevent them regardless of attack methods and vectors to protect various systems from even unknown zero-day exploits. </b>\n\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\tTo do so, we develop automated techniques that can analyze programs (in both binaries and source code) leveraging compiler-based techniques (e.g., <a href=\"https://llvm.org/\" target=\"_blank\">LLVM</a>) and reverse engineering (e.g., <a href=\"https://www.hex-rays.com/products/ida/index.shtml\" target=\"_blank\">IDA</a>), and secure the software by elliminating vulnerabilities from programs via instrumentations (e.g., binary rewriting <a href=\"data/revarm_acsac17.pdf\" target=\"_blank\">[ACSAC'17]</a>) <a href=\"data/a2c_ndss17.pdf\" target=\"blank\">[NDSS'17]</a><a href=\"data/pad_ase17.pdf\" target=\"_blank\">[ASE'17]</a>.\n\t\t\t\t\t\t\tWe also leverage SAT solvers (e.g., <a href=\"https://github.com/Z3Prover/z3/wiki/Documentation\" target=\"_blank\">Z3 solver</a>) to understand malicious payload and break them <a href=\"data/a2c_ndss17.pdf\" target=\"blank\">[NDSS'17]</a>.\n\t\t\t\t\t\t\t<br>\n\n\t\t\t\t\t\t\t<b>Examples include</b> (1) preventing zero-day attacks such as WannaCry without knowing the attack vector <a href=\"data/a2c_ndss17.pdf\" target=\"blank\">[NDSS'17]</a>, (2) protecting web users from malvertising attacks that exploit vulnerabilities in JavaScript/ActionScript engines <a href=\"data/pad_ase17.pdf\" target=\"_blank\">[ASE'17]</a>.\n\t\t\t\t\t\t</p>\n\t\t\t\t\t\t<br>\n\t\t\t\t\t\t<p>\n\t\t\t\t\t\t\t<b><font color=\"black\" size=\"4em\"><i class=\"fas fa-puzzle-piece\"></i> Reconstructing Attack Paths of Advanced Cyberattacks (e.g., <a href=\"https://en.wikipedia.org/wiki/Advanced_persistent_threat\" target=\"_blank\">Advanced Persistent Threats</a>)</font></b><br>\n\t\t\t\t\t\t\t<img src=\"images/research/aptattack.png\" style=\"border-radius: 8px;\" width=\"300\" id=\"flimg\">\n\t\t\t\t\t\t\tRecent cyber attacks become more and more sophisticated. In particular, advanced persistent threat or APT is a special kind of attack that leverages most advanced stealthy attack methods. They lurk in systems for a long time (e.g., weeks or months), infecting other systems through complex and large programs such as web-browsers.\n\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\t<b>\n\t\t\t\t\t\t\tMy research aims to precisely uncover attack paths of such advanced cyber attacks. I proposed a novel causal analysis technique <a href=\"data/ldx_asplos16.pdf\" target=\"blank\">[ASPLOS'16]</a> which precisely infers causality between system events (e.g., system calls) via input perturbation.\n\t\t\t\t\t\t\t</b>\n\t\t\t\t\t\t\tThe key insight of the technique is that rather than tracking information flow through instructions which often cause significant overhead and false positives/negatives, we run two executions of a program and perturb inputs that we want to track causality in one of the executions. Except those inputs, we maintain two executions identical. Hence, any output differences between two executions are caused by the perturbation, meaning that those differences and perturbations are causally related <a href=\"data/ldx_asplos16.pdf\" target=\"blank\">[ASPLOS'16]</a>.<br>\n\n\t\t\t\t\t\t\tMoreover, we develop a practical causality inference system for enterprise environment without any instrumentation requirements <a href=\"data/mci_ndss18.pdf\" target=\"_blank\">[NDSS'18]</a>.\n\t\t\t\t\t\t\t<br>\n\n\t\t\t\t\t\t\t<b>Examples include</b> (1) identifying information leak from malicious web-extensions <a href=\"data/ldx_asplos16.pdf\" target=\"blank\">[ASPLOS'16]</a> [<a href=\"https://drive.google.com/file/d/0B01NFmIJ403gSnhRQWFVMjNPM1U/view?usp=sharing\" target=\"_blank\">Demo Video</a>], (2) reconstructing the entire attack story from the initial compromise and system disruptions in enterprise environments <a href=\"data/mci_ndss18.pdf\" target=\"_blank\">[NDSS'18]</a>, (3) understanding and revealing details of library hijacking attacks <a href=\"data/lprov_acsac18.pdf\" target=\"_blank\">[ACSAC'18]</a>.\n\n\n\t\t\t\t\t\t</p>\n\t\t\t\t\t\t<br>\n\t\t\t\t\t\t<p>\n\t\t\t\t\t\t\t<b><font color=\"black\" size=\"4em\"><i class=\"fas fa-search\"></i> Analyzing Malicious Binaries via Binary Analysis and Reverse-Engineering (across Multiple Platforms) </font></b><br>\n\t\t\t\t\t\t\t<img src=\"images/research/radare.png\" style=\"border-radius: 8px;\" width=\"400\" id=\"flimg\">\n\t\t\t\t\t\t\tAnalyzing malicious binary programs is difficult because, in part, these binaries are obfuscated and/or input/environment sensitive. Moreover, recent attacks happen across multiple platforms while tools that can analyze malicious binaries often do not support such new platforms.\n\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\t<b>\n\t\t\t\t\t\t\tMy research aims to analyze malicious binaries via dynamic analysis and program transformation.\n\t\t\t\t\t\t\twe proposed a novel program transformation technique which can transform a platform-dependent program execution (e.g., an IoT malware execution) into a platform-independent program so that\n\t\t\t\t\t\t\tit can be analyzed by existing program analysis tools <a href=\"data/pietrace_ase13.pdf\" target=\"blank\">[ASE'13]</a>.\n\t\t\t\t\t\t\tAlso, we reverse engineer malicious binaries to discover message/file formats of various malicious software such as botnets <a href=\"data/p2c_ndss15.pdf\" target=\"blank\">[NDSS'15]</a>.\n\t\t\t\t\t\t\t</b>\n\t\t\t\t\t\t\t<br>\n\t\t\t\t\t\t\tAlso, we develop binary rewriting infrastructures <a href=\"data/revarm_acsac17.pdf\" target=\"_blank\">[ACSAC'17]</a> which can insert/remove functionalities in existing binary programs without access to source code.\n\t\t\t\t\t\t\t<br>\n\n\t\t\t\t\t\t\t<b>Examples include</b> (1) analyzing vulnerabilities on a sensor platforms via x86 Linux program analysis tools (e.g., <a href=\"https://en.wikipedia.org/wiki/Program_slicing\" target=\"_blank\">program slicing</a> tools) <a href=\"data/pietrace_ase13.pdf\" target=\"blank\">[ASE'13]</a>, (2) reverse-engineering Command and Control protocol of a botnet and data leaked by the botnet with no prior knowledge on the botnet <a href=\"data/p2c_ndss15.pdf\" target=\"blank\">[NDSS'15]</a>, (3) adding various protections (e.g., <a href=\"https://en.wikipedia.org/wiki/Control-flow_integrity\" target=\"_blank\">Control Flow Integrity</a>, <a href=\"https://en.wikipedia.org/wiki/Buffer_overflow_protection\" target=\"_blank\">Buffer overrun protection</a>) to existing ARM binaries and reducing unnecessary binary code from the binary program directly without source code <a href=\"data/revarm_acsac17.pdf\" target=\"_blank\">[ACSAC'17]</a>.\n\n\n\n\t\t\t\t\t\t</p>\n                      </div>\n                      <!-- /research Content -->\n                    </div>\n                  </div>\n
