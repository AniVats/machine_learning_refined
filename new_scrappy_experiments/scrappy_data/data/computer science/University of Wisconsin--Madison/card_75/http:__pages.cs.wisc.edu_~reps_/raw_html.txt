name=\"codesurfer\">\n</a>\n<p>\nThe Wisconsin program-slicing technology is now available in a\ncommercial product, the\n<a href=\"http://www.grammatech.com/products/codesurfer/codesurfer.html\">CodeSurfer</a><small><sup><sup>tm</sup></sup></small>\ntool available from <a href=\"http://www.grammatech.com\">GrammaTech, Inc.</a>\n</p><p>\nCodeSurfer builds a dependence-graph program representation\nand provides a GUI for exploring this web.\nThe dependence graph includes forward and backward links between each\nassignment statement and possible uses of the values stored by that\nassignment.\nPointer analysis is used so that indirect loads and stores through\npointers are taken into account, as well as indirect function calls.\nDataflow analysis is used so that links between unrelated\nassignments and uses are excluded.\nOperations that highlight forward and backward slices show the impact\nof a given statement on the rest of the program (forward slicing), and\nthe impact of the rest of a program on a given statement (backward\nslicing) [<a href=\"#toplas90\">TOPLAS90</a>, <a href=\"#fse94\">FSE94</a>].\nOperations that highlight paths between\nnodes in the dependence graph (chops) show ways in which the\nprogram points are interdependent (or independent)\n[<a href=\"#fse95b\">FSE95b</a>].\nCodeSurfer's scripting language, which provides access to the\ndependence-graph program representation and the Tk widget set, is used\nfor extensibility and for batch applications.\n</p><p>\nFull information about CodeSurfer is available\n<a href=\"http://www.grammatech.com/products/codesurfer/codesurfer.html\">here</a>, where you can find\n</p><ul>\n  <li> full
