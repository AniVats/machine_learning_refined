current projects involve a novel secure operating system and system infrastructure\nfor high-performance cloud computing applications.\nI like building things, especially\nsystems that have some principled reason for why they should work.\n</p>\n\n<p>\nHere is some information for <a href=\"prospectives.php\">prospective students</a> interested in working with me.\n</p>\n\n<table width=\"95%\" border=\"0\">\n<tbody><tr><td colspan=\"2\"><h3>Recent Work</h3></td></tr>\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"http://hyperdex.org/\">HyperDex</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\nHyperDex is a new NoSQL key-value store. It is fast, consistent, and fault tolerant, with a rich API that includes an efficient search primitive. It is significantly faster than Cassandra, MongoDB and Redis; scales well; and provides a level of consistency and fault tolerance not found in other systems. \n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"http://openreplica.org/\">OpenReplica</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\nOpenReplica is a service for replicating objects based on our ConCoord Paxos imlementation for replica coordination. The OpenReplica service enables anyone to quickly and easily replicate objects, to locate replicas through DNS, and to dynamically modify the replica set. It's comparable to Yahoo's ZooKeeper and Google's Chubby, except we run the system as a service, therefore anyone can deploy replicated objects simply by uploading a Python object. OpenReplica achieves performance that is comparable to or better than ZooKeeper for less than 6 replicas.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"nexus/\">Nexus</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    I am building a new operating system called Nexus. Nexus introduces a new driver architecture, new abstractions and new mechanisms that enable secure, trustworthy applications. The system boots standalone on x86 platforms ideally equipped with TPMs; it can send mail, play movies, run cloud applications and execute Linux programs.\n   </td></tr>\n\n   <tr>\n   <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"http://frenetic-lang.org/merlin/\">Merlin</a></td>\n   <td valign=\"TOP\" colspan=\"1\"> \nMerlin is a new network management framework that allows administrators to express policies in a high-level, declarative language.\nOverall, Merlin simplifies the task of network administration by providing high-level abstractions for specifying network policies and scalable infrastructure for enforcing them.\n    </td></tr>\n\n\n<tr><td colspan=\"2\"><h3>Peer-to-peer Systems</h3></td></tr>\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"http://www.cs.cornell.edu/~bwong/cubit/\">Cubit</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Cubit is a peer-to-peer overlay that enables approximate searches in large networks with no centralized components. Given a potentially misspelled keyword, Cubit finds all objects containing that keyword. The project was inspired by the various legal attacks and attempts to take down torrent sites. The key idea behind Cubit is to construct a metric space for keywords, map it onto nodes in a small world, and then to traverse the nodes using a greedy routing algorithm. We have implemented Cubit as an Azureus plugin.\n\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"http://www.cs.cornell.edu/~bwong/octant/\">Octant</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Octant is a system for determining the physical location of Internet hosts. Given a host, Octant determines the boundaries of the region in which the node is likely to lie. Behind the scenes, Octant consists of two parts: a comprehensive framework for efficiently representing and combining a system of constraints, and a set of mechanisms for extracting tight constraints on node locations without resulting in\nan overconstrained system.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"meridian/\">Meridian</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Meridian is a peer-to-peer overlay network for performing location-aware node and path selection in large-scale distributed systems. It is simple to deploy, robust to\nchurn, and can accurately find the nearest node, pick the most centrally placed node, and find a node that fits latency constraints.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"beehive/\">Beehive CoDoNS CobWeb CorONA</a>\n    </td>\n    <td valign=\"TOP\" colspan=\"1\">\n    <p class=\"tbl\">\n    Beehive is a high-performance distributed hash table. A novel\n    optimization technique enables Beehive to respond to queries \n    quickly, tolerate\n    denial of service attacks, and balance load. Beehive provides\n    strong performance guarantees even in the presence of queries\n    drawn from Power Law distributions, previously thought to be\n    difficult because heuristics-based approaches tried in the past do not\n    work well with such distributions.\n\n    </p><p class=\"tbl\">\n    We have used \n    Beehive to build new, resilient infrastructure services for the\n    Internet. <a href=\"beehive/codons.php\">CoDoNs</a> is a \n    safety net and a replacement for the Domain Name System that \n    provides strong security, performance, and fast \n    dynamic updates for existing Internet names. \n    <a href=\"beehive/cobweb/\">CobWeb</a> is an Akamai-like\n    open-access content distribution network. \n    <a href=\"beehive/corona/\">CorONA</a> is a high-performance publish-subscribe system for web micronews. </p>\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"http://www.cs.cornell.edu/People/egs/sqrt-s/\">Sqrt(S)\n</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n<img src=\"sqrt-s/pic/s2B.jpg\" border=\"0\" alt=\"sqrt-s\" width=\"20\" height=\"20\">\n is a practical and efficient failure detector.\nIt is particularly suitable for systems where a node is tasked with monitoring the liveness status of a \nlarge number of other nodes, such as peer-to-peer systems.\nIt determines how often to monitor each node such that failure detection latency\nis minimal while bandwidth usage remains below a user-specified limit. Unlike \nmost other work on theoretical failure detectors, it is not only usable in real systems, but also simple and efficient.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"credence/\">Credence</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Credence is a reputation system for peer-to-peer networks, designed to \n    provide an accurate metric for the trustworthiness of labels associated\n    with shared files. It differs from previous work in that it derives\n    its trust metric from principled measures that reflect likelihood\n    of similar behavior between peers, has a completely\n    distributed architecture with no trusted nodes, and a concrete \n    implementation. Credence can guard against Sybil attacks and \n    other malicious behavior from\n    spammers. The Credence implementation is free, open-source,\n    and backwards compatible with Gnutella.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"corsso/\">CorSSO</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    CorSSO is a distibuted authentication service that provides\n    network identities that span multiple application services, also\n    known as single sign-on. It enables authentication functionality\n    to be factored out of application services and delegated to\n    combinations of authentication servers. It uses threshold\n    cryptography for efficiency, fault tolerance and resiliance\n    against attackers.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"karma/\">Karma</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Karma is a virtual currency for use in keeping track of users'\n    resource contribution and consumption in peer-to-peer systems.\n    The karma system provides a secure exchange mechanism for a\n    self-regulating, incentive-compatible, decentralized currency.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"herbivore/\">Herbivore</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Herbivore is a peer-to-peer, self-organizing, robust system for\n     anonymous communication. It uses dining cryptographer networks\n    to provide anonymity guarantees even in the presence of attackers\n    which can eavesdrop on every packet in the network.\n    It's a follow-on to the \n     <a href=\"cliquenet/\">CliqueNet</a> project.\n\n</td></tr><tr><td colspan=\"2\"><h3>Operating Systems and Virtual Machines</h3></td></tr>\n\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"nexus/\">Nexus</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    I am building a new operating system called Nexus. Nexus introduces a new driver architecture, new abstractions and new mechanisms that enable secure, trustworthy applications. The system boots standalone on x86 platforms ideally equipped with TPMs; it can send mail, play movies, run cloud computing applications and execute Linux programs.\n\n</td>\n  </tr>\n\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"http://www.cs.cornell.edu/~ashieh/trickles/\">Trickles</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Trickles is a high-performance protocol for stateless, connection-oriented\n    communication. It comprises a transport protocol to replace TCP and a\n    new interface to replace sockets that allow all server-side state\n    to be shipped to clients. This leads to applications that are\n    more scalable and robust against denial-of-service attacks. And it\n    enables a new class of services:\n    since a stateless network stack allows packets to be serviced at \n    any server replica regardless of past history, Trickles enables \n    fast transparent failover, fine-grain load balancing and \n    connection-oriented anycast services to be implemented\n    transparently inside the network fabric.\n</td>\n  </tr>\n\n\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"portos/\">Portos</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Portos is an emulation-based system and a set of corresponding \n    projects that I use to teach introductory-level\n    operating systems. The projects cover traditional topics, as well as new issues (such as routing and self-organization) raised by mobile, ubiquitous computing. The base emulation framework emulates a virtual processor, with attached \nvirtual devices, on Windows NT (including 98/ME/XP/etc.) and CE. It runs on x86 and StrongArm processors. It's freely available.\n</td>\n  </tr>\n\n\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"kimera/\">Kimera</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\nThe goal of the Kimera project is to enable networks of computers that\nare cheaper, more secure and more manageable than what we have\nnow. The problem with current state of the art systems, like Java, is that network clients are monolithic, that is, they implement all requisite\nsystem services locally. Consequently, every endpoint needs to\nhave sufficient resources to support services like verification,\ncompilation and security management, which require too much memory\nand processing power for embedded devices. Further, each endpoint\nentails associated software state that is hard to secure and manage. \nThese problems become particularly acute as the number of clients\nincreases.\n<p>\nThe Kimera project addresses these problems by factoring system\ncomponents out of clients into network servers. The clients can thus\nbe smaller because they do not have to support complex services locally. \nFurther, the overall network is easier to manage because service \nfunctionality is centralized. The services operate by intercepting\napplications as they are fetched by clients; they inspect applications, and where necessary, inject code snippets into them to provide the requisite service\nfunctionality. Digital signatures ensure that only applications that\nhave been vetted by centralized services execute on the clients. \n\n</p></td>\n  </tr>\n  <tr>\n    <td valign=\"TOP\"><a href=\"http://www.cs.washington.edu/research/projects/spin/www/\">SPIN</a>\n    </td>\n    <td valign=\"TOP\">\nThe goal of the SPIN project is to enable applications to customize\noperating system functionality by downloading application-specific\nextensions dynamically into the kernel. The extensions, as well as the\ncore system, are written in Modula-3, whose type-safety ensures load\nand store protection. A namespace management interface restricts the\ninterfaces that applications can access, and creates multiple\nsoftware protection domains within a single privileged address space. \n\n<p>\nThe SPIN execution model enables applications to safely extend the \nkernel with thread packages and hierarchical scheduling policies. A\ncombination of language and system mechanisms provide security and\nfault-isolation while mediating access to processors. The SPIN\nprotection domain interface simultaneously allows isolation and safe\nfine-grain sharing within a privileged address space. Extensions\nreside in hierarchical namespaces which they can use to share or to\nhide code and data at the granularity of interfaces. Use of\nhierarchical capabilities simplifies security management. Extensions\nthat want to safely share code and data can do so without dynamic\nprotection enforcement overhead. The SPIN web server uses all of these\nmechanisms to implement high-performance web service. \n</p></td>\n</tr>\n<tr>\n<td valign=\"TOP\"><a href=\"mipsi/mipsi.html\">MIPSI</a></td>\n<td>\nMIPSI is a robust and extensive MIPS instruction set simulator.\nIt has been used in many classes and research\nprojects.\n</td>\n  </tr>\n\n<tr><td colspan=\"2\"><h3>Ad Hoc Networks</h3></td></tr>\n  <tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"magnetos/\">MagnetOS</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    MagnetOS is an operating system for ad hoc networks. It makes the entire\n    network appear as a single Java virtual machine. It enables applications\n    to be constructed easily and to execute efficiently.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\" width=\"8%\"><a href=\"sextant/\">Sextant</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    Sextant is a general framework for discovering the location of\n    nodes and events in wireless networks. Sextant enables nodes without\n    specialized hardware, such as GPS, to efficiently discover their\n    approximate locations. Further, given a sensor network where each node\n    can sense events in its immediate vicinity, Sextant pinpoints the\n    location of events with high fidelity.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"sharp/\">SHARP</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    SHARP is a hybrid routing protocol that dynamically finds the optimal\n    mix of proactive route dissemination and reactive route discovery to\n    achieve application-specific performance goals.\n\n  </td></tr><tr>\n    <td valign=\"TOP\" rowspan=\"1\"><a href=\"sns/\">SNS</a></td>\n    <td valign=\"TOP\" colspan=\"1\">\n    SNS is a scalable, high-performance wireless network simulator, \n    based on ns2. It vastly outperforms standard ns2 in speed and scale through a new technique we developed called <i>staged simulation</i>. It achieves\nits improvements in speed and scale by eliminating redundant computations both\nwithin a single and across multiple simulation runs. It has been carefully\nvalidated against ns2 - staging preserves accuracy while speeding up simulations.\n   </td>\n  </tr>\n\n</tbody></table>\n<p>\n\n</p><h2>Teaching</h2>\n\n<table
